/ PRESENTATION

Project Presentation

INTRODUCTION · METHODOLOGY · CONCLUSION · REFERENCES

SLIDE 01 Introduction
  • Automation reduces manual effort and errors
  • Manual object sorting is inefficient
  • Vision-based robotics combines camera + robotic arm
  • Project focuses on sorting stationary objects by color, shape, and size
SLIDE 02 Motivation
  • Automation is increasingly used in industries to improve efficiency and reduce human effort
  • Manual sorting of objects is time-consuming and prone to errors
  • Robotic arms like the Dobot Magician can perform precise pick-and-place tasks
  • Integrating computer vision (OpenCV) with robotics enables intelligent object detection and sorting
  • This project demonstrates an automated color-based sorting system using a robot and vision processing
METHODOLOGY
SLIDE 03 Selected Serial Manipulator
  • Dobot Magician — 4-DOF serial manipulator for precision pick-and-place
  • Revolute Joint 1 (θ₁) — Base Rotation: enables horizontal rotation
  • Revolute Joint 2 (θ₂) — Shoulder Joint: controls vertical lifting of primary arm link
  • Revolute Joint 3 (θ₃) — Elbow Joint: controls arm extension and reach
  • Revolute Joint 4 (θ₄) — Wrist/End-Effector: adjusts gripper orientation
  • End Effector — Suction Cup for grasping and releasing objects
  • Modeled using Denavit–Hartenberg (DH) parameters for forward and inverse kinematics
SLIDE 04 Simulation Environment Setup
  • Developed using CoppeliaSim — a physics-based robotic simulation platform
  • Dobot Magician robotic arm model imported into the scene
  • Conveyor belt system added to transport objects
  • Multiple colored objects placed on the conveyor
  • Vision sensor (camera) positioned above the conveyor
  • Proximity sensor placed near the robot workspace
  • Gravity, object dynamics, and workspace boundaries configured for realistic operation
SLIDE 05 Python–Simulation Communication
  • Connected via ZeroMQ Remote API provided by CoppeliaSim
  • Python scripts can read proximity sensor data and capture images from vision sensor
  • Controls conveyor velocity, robot joints, and suction gripper activation
  • Process: establish connection → retrieve object handles → send motion commands → receive sensor feedback
  • Enables seamless integration of computer vision with robotic motion control
SLIDE 06 Robot Configuration & Initialization
  • Joint handles retrieved from the simulation
  • Initial joint angles recorded
  • Velocity, acceleration, and jerk limits defined for smooth motion
  • Custom smooth motion control function implemented
  • Predefined Home Position assigned — safe standby before object detection
SLIDE 07 Conveyor Belt & Object Detection
  • Conveyor motor activates → objects move toward detection zone
  • Proximity sensor detects object using signal reflection
  • Detection signal sent to Python → conveyor stops immediately
  • Object remains stationary for image acquisition and processing
  • Guarantees stable and accurate image capture
SLIDE 08 Image Acquisition & Processing
  • Image captured from vision sensor → transferred to Python → converted to NumPy array
  • Flipped vertically (sensor output is inverted) → converted from RGB to BGR for OpenCV
  • Step 1: BGR → HSV; separates color from brightness for robust detection
  • Step 2: Predefined HSV ranges create binary masks isolating Red and Blue objects
  • Step 3: Median filtering removes noise and improves contour quality
  • Step 4: Contour detection extracts object area, boundary, and centroid
SLIDE 09 Shape Detection
  • Shape identified using polygon approximation on detected contours
  • Calculate perimeter → apply Douglas–Peucker approximation → count vertices
  • 3 Vertices Triangle
  • 4 Vertices Square (aspect ratio 0.96–1.05)
  • > 6 Vertices Circle
  • Enables geometry-based classification alongside color detection
SLIDE 10 Inverse Kinematics & Pick Operation
  • IK computes joint angles θ₁–θ₄ for end-effector to reach target position
  • Defined positions: Home → Approach → Pick → Drop
  • Robot moves to Approach Position → descends to Pick Position → suction gripper activates
  • Object lifted back to safe approach height to prevent collisions
SLIDE 11 Sorting, Placement & Continuous Operation
  • Drop location determined by detected object properties
  • ● RED → Bin 1    ● BLUE → Bin 2    △ TRIANGLE → Bin 3
  • Robot lowers object → suction deactivates → robot retracts safely
  • Robot returns to Home Position → conveyor restarts → system awaits next object
  • Full cycle: Detect → Capture → Classify → Pick → Sort → Place

SLIDE 13 Conclusion
  • Accurate detection of color and shape achieved through HSV thresholding and polygon approximation
  • The suction-based end effector was not able to reliably grip smaller objects, which limited the system’s ability to perform size-based sorting.
  • Reliable rule-based classification enables consistent sorting decisions
  • Smooth simulated robotic arm movement validated using CoppeliaSim with IK-based motion control
  • System validated through online simulation prior to hardware deployment

/ OPENCV CODE

OpenCV Color, Shape & Size Detection

PYTHON · OPENCV · HSV · CANNY EDGES · DOUGLAS–PEUCKER · REAL-TIME

object_detection.py 
import cv2
import numpy as np

# ==========================
# PARAMETERS (USER DEFINED)
# ==========================

MIN_AREA = 1500        # min_area from paper
MAX_AREA = 50000       # max_area from paper
SIZE_THRESHOLD = 7000  # small vs big (tune once)

# ==========================
# CAMERA INITIALIZATION
# ==========================

cap = cv2.VideoCapture(0)

if not cap.isOpened():
    print("Error: Camera not accessible")
    exit()

# ==========================
# MAIN LOOP
# ==========================

while True:
    ret, frame = cap.read()
    if not ret:
        break

    frame = cv2.resize(frame, (640, 480))

    # --------------------------------------------------
    # (i) RGB → HSV conversion
    # --------------------------------------------------
    hsv = cv2.cvtColor(frame, cv2.COLOR_BGR2HSV)

    # --------------------------------------------------
    # (ii) Lower & Upper HSV values
    # --------------------------------------------------

    # RED (two ranges)
    lower_red1 = np.array([0,   100, 100])
    upper_red1 = np.array([10,  255, 255])
    lower_red2 = np.array([170, 100, 100])
    upper_red2 = np.array([180, 255, 255])

    # BLUE
    lower_blue = np.array([100, 100, 100])
    upper_blue = np.array([140, 255, 255])

    # --------------------------------------------------
    # (iii) Mask image creation
    # --------------------------------------------------
    mask_red  = cv2.inRange(hsv, lower_red1, upper_red1) +                 cv2.inRange(hsv, lower_red2, upper_red2)
    mask_blue = cv2.inRange(hsv, lower_blue, upper_blue)

    # Process each color separately
    for mask, color_name, color_bgr in [
        (mask_red,  "Red",  (0, 0, 255)),
        (mask_blue, "Blue", (255, 0, 0))
    ]:
        # (i) Noise removal using median filter
        mask = cv2.medianBlur(mask, 5)

        # (iv) Canny edge detection
        edges = cv2.Canny(mask, 100, 200)

        # (viii) Contour detection (boundary fill equivalent)
        contours, _ = cv2.findContours(
            edges, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE
        )

        for cnt in contours:
            # (v) Area computation (Green's theorem equivalent)
            area = cv2.contourArea(cnt)

            # (vi) min_area & max_area check
            if area < MIN_AREA or area > MAX_AREA:
                continue

            # (vii) Centroid computation
            M = cv2.moments(cnt)
            if M["m00"] == 0:
                continue
            cx = int(M["m10"] / M["m00"])
            cy = int(M["m01"] / M["m00"])

            # SIZE CLASSIFICATION (PROJECT EXTENSION)
            size = "Small" if area < SIZE_THRESHOLD else "Big"

            # (ix) Douglas–Peucker Algorithm
            perimeter = cv2.arcLength(cnt, True)
            epsilon   = 0.02 * perimeter
            approx    = cv2.approxPolyDP(cnt, epsilon, True)

            # (x) & (xi) Shape Detection
            shape = "Unknown"
            if len(approx) == 3:
                shape = "Triangle"
            elif len(approx) == 4:
                x, y, w, h = cv2.boundingRect(approx)
                ar = w / float(h)
                if 0.96 < ar < 1.05:
                    shape = "Square"
                else:
                    continue  # ignore rectangles
            elif len(approx) > 6:
                shape = "Circle"
            else:
                continue

            # DRAW RESULTS
            x, y, w, h = cv2.boundingRect(cnt)
            cv2.rectangle(frame, (x, y), (x+w, y+h), color_bgr, 2)
            cv2.circle(frame, (cx, cy), 4, (0, 255, 0), -1)
            label = f"{color_name}, {shape}, {size}"
            cv2.putText(frame, label, (x, y - 10),
                        cv2.FONT_HERSHEY_SIMPLEX, 0.6, color_bgr, 2)

    # DISPLAY
    cv2.imshow("Paper-Aligned Object Detection", frame)
    if cv2.waitKey(1) & 0xFF == 27:  # ESC to quit
        break

# ==========================
# CLEANUP
# ==========================
cap.release()
cv2.destroyAllWindows()
